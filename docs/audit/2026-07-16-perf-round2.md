## Remaining / deferred (verified real, not yet applied)

Ranked by value. All are larger/architectural with real behavior tradeoffs —
each is a deliberate task, not a drive-by.

### High value
- ~~**cacheComponents route-group split**~~ ✅ **SHIPPED 2026-07-16.** Auth +
  realtime stack scoped to the `(app)` route group (`8a8f7be`); `cacheComponents:
  true` enabled → all routes now `◐ Partial Prerender` (static shell + streamed
  data), public /share·/site·/forms no longer boot the Convex client. Follow-up:
  per-route `"use cache"` data caching (privacy nuance for /share) —
  `docs/audit/cache-components.md`. ⚠️ experimental flag — smoke-test the deploy.
- **`pages.listMeta` ships `rowProps` for the whole workspace** (P2, large,
  `convex/pages.ts:210`). Every block save re-broadcasts every DB row's props
  to the sidebar sub. Fix: drop `rowProps` from listMeta + a dedicated
  per-open-DB `rowsForDatabase(dbId)` sub for `useRows`. High blast radius
  (NotionDatabase cell render, csv/table export, relation mirrors, snapshots).

### Medium
- **Admin analytics live subscriptions** (P2, large, `convex/admin/queries.ts`).
  `getOverview` = 8 full-table `.take(25000)` scans subscribed live; any write
  to any of the 8 tables re-runs all 8. Admin-only. Fix: non-reactive
  `fetchQuery` / cron-refreshed stats singleton.
- **Dashboard content routes have no server preload** (P2, large,
  `app/dashboard/p/[id]`, `db/[id]`). First paint gated behind whole-workspace
  store load + global splash. Fix: `preloadQuery`/`usePreloadedQuery` per doc
  (needs SSR auth-token wiring) and/or drop the global `isInitialLoading` gate.
- **Full table/page virtualization** (P2→P3). `content-visibility` on blocks
  shipped (partial). True react-virtual windowing regresses dnd-kit drag,
  marquee-select, keyboard nav, sticky frozen columns — needs a browser-tested
  dedicated task; only pays off at 1000+ rows / 500+ blocks (uncommon here).
- **`pages.getById` over-fetch** (P3, `:253`). Hot editor sub spreads the full
  doc incl. `searchText` (≤8KB dup of block text) + tags/titleKey/blockCount/
  previewText that no consumer reads. Fix: explicit DTO (enumerate ~24 kept
  fields carefully — `layouts` is read via a cast, so tsc won't catch a drop).

### Low
- StoreCtx action-bundle memo / context split (P3) — monolithic context churn;
  cheap memo has low ROI, real win is a context split (large).
- `katex` static import on `/share` (P3) — CSS ships to no-math pages; fix must
  keep CSS for inline `$math$` + preserve SSR.
- `vaul` (~11KB gzip) first-load, desktop-unused (P3) — lazy the drawer subtree.
- Tweakcn switcher panel lazy (P3, low ROI — the lib core is legitimately eager).
- `proxy.ts` matcher narrowing (P3) — behavior change (auth cookie refresh on
  public-only browsing); needs auth-flow testing.
- `snapshots.listAll` is now used only on-demand by the backup export — a later
  convex deploy could inline/remove it if the export switches to a paginated
  read.
